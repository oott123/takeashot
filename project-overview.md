# Takeashot 项目总结

## 1. 项目概述

Takeashot 是一款专为 KDE Plasma Wayland 环境设计的截图工具。它提供交互式的区域选择界面，支持多显示器、HiDPI、窗口吸附、标注绘制和剪贴板集成。

项目仅支持 KDE Wayland，这是一个有意为之的决策——利用 KWin 提供的 D-Bus 截图接口和脚本 API，可以获得原生级别的屏幕捕获和窗口信息，而无需依赖通用但受限的跨桌面方案。

---

## 2. 启动与触发

应用常驻后台，以系统托盘图标的形式存在。触发截图有两种方式：

- **按下 Pause 键**：全局监听，无论当前焦点在哪个应用都能响应。
- **再次启动应用**：如果已有实例在运行，新启动的进程不会创建第二个窗口，而是通知已有实例立即进入截图模式。

进入截图模式后，每个显示器上都会覆盖一层半透明的暗色遮罩，屏幕内容被"冻结"为截图瞬间的画面。

---

## 3. 选区

### 3.1 三种选区状态

截图模式下，系统始终处于以下三种状态之一：

**无选区**：初始状态。所有屏幕被半透明遮罩覆盖，用户可以自由移动鼠标。

**拟选择（Pending）**：当窗口吸附功能可用时，鼠标悬停在某个窗口上会自动出现蓝色边框预览，表示"如果你现在点击，就会选中这个区域"。此时没有调整手柄，仅仅是一个预览。窗口按照前后层叠顺序匹配——鼠标下方最前面的窗口优先。

**有选区**：选区已确定。遮罩在选区范围内被"挖空"，露出下方的原始截图。选区四周和四角共显示 8 个调整手柄，工具栏也随之出现。

### 3.2 创建选区

有两种方式创建选区：

- **拖拽**：在任意位置按下鼠标并拖动，松开后形成矩形选区。
- **点击窗口预览**：在拟选择状态下单击，直接将窗口区域确认为选区。

如果在拟选择状态下不是点击而是拖拽，则忽略窗口预览，以拖拽的矩形为准。

### 3.3 调整选区

选区确定后可以进行多种调整：

- **移动**：在选区内部拖拽，整个选区跟随移动。
- **调整大小**：拖拽 8 个手柄中的任意一个，改变选区的边界。
- **外部扩展**：在选区外部点击，选区会朝点击方向扩展以包含该点。如果点击位置在选区的水平范围内，则只纵向扩展；在垂直范围内则只横向扩展；在对角方向则同时扩展两个方向。在选区外部拖拽则类似于拖拽一个虚拟手柄，可以持续调整大小。

### 3.4 跨屏幕选区

选区可以跨越多个显示器。每个显示器的窗口只渲染属于自己屏幕的那部分选区。确认截图时，各屏幕的内容会被拼合成一张完整的图像。

### 3.5 确认与取消

- **确认**：按下 Enter 键或在选区内双击，截图（含标注）被复制到剪贴板，截图模式退出。
- **取消**：按下 Esc 或右键单击，拟选择状态直接退出，有选区状态退回无选区状态。

---

## 4. 光标

光标形状会根据鼠标位置和当前状态动态变化，为用户提供操作提示：

- **十字准星**：无选区时，或鼠标在选区外部且未处于拟选择状态。表示"可以在此开始拖拽创建选区"。
- **移动光标（四向箭头）**：鼠标在选区内部，且当前工具为移动工具。表示"拖拽可以移动选区"。
- **调整大小光标（双向箭头）**：鼠标悬停在某个调整手柄上，箭头方向与手柄的调整方向一致（水平、垂直或对角）。在选区外部拖拽调整时，光标也会切换为对应方向的双向箭头。
- **十字准星（选区内）**：当前选择了标注绘制工具时，即使在选区内部，光标也显示为十字准星而非移动光标，表示"拖拽将绘制标注而非移动选区"。

---

## 5. 工具栏

当选区存在时，工具栏自动出现。工具栏是一个白底黑边的横条，包含一排图标按钮（使用 Tabler Icons）。

### 5.1 定位策略

工具栏的位置按优先级依次选择：
1. 选区右下角的外侧下方（首选，不遮挡选区内容）
2. 选区右上角的外侧上方（当下方空间不足时）
3. 选区内部的右下角（当上下都没有足够空间时）

工具栏会随着选区的移动和调整而重新定位。

### 5.2 工具列表

工具栏提供以下工具，同一时间只能激活一个：

**移动选区（默认）**：鼠标操作作用于选区本身——拖拽移动、手柄调整大小、外部扩展。不能操作标注。

**标注编辑**：切换到标注操作模式。可以点击选中已有标注，拖拽移动标注位置，拖拽角点手柄缩放，拖拽旋转手柄旋转。不能操作选区。切换到其他工具时自动取消标注的选中状态。

**画笔**：按住鼠标在选区内拖动，绘制自由笔迹。

**直线**：从按下位置到松开位置绘制一条直线。

**矩形**：拖拽绘制矩形。

**椭圆**：拖拽绘制椭圆。

### 5.3 鼠标事件穿透

工具栏本身会遮挡一部分屏幕区域。工具栏按钮之外的空白区域（如工具栏上方的 tooltip 空间）不拦截鼠标事件，而是将其透传给下方的截图组件，避免出现"点不到"的死区。

---

## 6. 标注

### 6.1 绘制

选择画笔、直线、矩形或椭圆工具后，在选区内拖拽即可绘制。标注仅在选区范围内可见。

### 6.2 编辑

切换到标注编辑工具后：
- 点击某个标注将其选中，出现角点手柄（缩放）和旋转手柄。
- 拖拽标注主体可移动位置。
- 拖拽角点手柄可缩放。
- 拖拽旋转手柄可旋转。
- 选中状态下按 Delete 可删除该标注。

### 6.3 合成输出

确认截图时，所有标注会与截图内容合成为一张图像后复制到剪贴板。标注不是单独的图层——用户从剪贴板粘贴后得到的就是带标注的最终图片。

### 6.4 生命周期

每次进入截图模式时，上一次的标注会被清除。标注只在当前截图会话中存在。

---

## 7. 多显示器与 HiDPI

每个显示器有独立的截图窗口，各自渲染属于自己屏幕的那部分内容。截图时各显示器并行捕获，以缩短截图延迟。

HiDPI 屏幕会按照物理像素捕获截图（而非逻辑像素），保证输出图像的清晰度。各显示器可以有不同的缩放比例。

---

## 8. 窗口吸附

进入截图模式后，应用会异步向 KWin 请求当前所有窗口的位置和大小信息。窗口按照 KWin 的层叠顺序排列——最前面（最近使用）的窗口优先匹配。

当数据就绪后，用户在无选区状态下移动鼠标，系统会自动找到鼠标下方最前面的窗口并显示蓝色边框预览。用户单击即可将该窗口区域设为选区，无需手动对齐边界。

如果窗口列表获取失败或超时，窗口吸附功能会静默禁用，不影响手动拖拽选区的使用。

---

## 9. 外部接口

本项目与 KDE/系统的交互全部通过以下外部接口完成。

### 9.1 截图捕获：KWin ScreenShot2 D-Bus

通过 Session Bus 调用 KWin 提供的截图接口：

- **服务**: `org.kde.KWin`
- **对象路径**: `/org/kde/KWin/ScreenShot2`
- **接口**: `org.kde.KWin.ScreenShot2`

使用两个方法：

- `CaptureScreen(screen_name, options, fd)` — 捕获指定显示器。`screen_name` 是显示器名称（如 `"DP-1"`），`options` 传入 `{'native-resolution': True}` 以获取物理像素分辨率的截图，`fd` 是一个临时文件的文件描述符，KWin 会将截图原始数据写入该 fd。
- `CaptureWorkspace(options, fd)` — 捕获整个工作区（所有显示器拼合），作为逐屏捕获失败时的回退方案。

返回值是一个 metadata 字典，包含 `width`、`height`、`stride`（每行字节数）、`format`（像素格式）。截图数据为 BGRA 格式的原始像素。

使用临时文件而非管道传输数据，是因为截图数据量大（多屏 HiDPI 可达数十 MB），管道容易因缓冲区满而死锁。

**权限要求**：Wayland 下此接口受限，调用方需要在 `~/.local/share/applications/` 下注册一个 `.desktop` 文件，并声明 `X-KDE-DBUS-Restricted-Interfaces=org.kde.KWin.ScreenShot2`。

### 9.2 窗口列表获取：KWin Scripting API

获取窗口信息的流程比截图复杂，因为 KWin 没有直接的"列出所有窗口"D-Bus 方法，需要通过 KWin 的脚本引擎间接实现：

1. **加载脚本**：调用 `org.kde.KWin` 的 `/Scripting` 对象（接口 `org.kde.kwin.Scripting`），方法 `loadScript(path, pluginName)`，传入一个临时 JavaScript 文件的路径。返回一个 `scriptId`。

2. **运行脚本**：调用 `/Scripting/Script{scriptId}` 对象（接口 `org.kde.kwin.Script`），方法 `run()`。

3. **脚本内容**：JavaScript 在 KWin 进程内执行，可以访问 `workspace.stackingOrder`（窗口按层叠顺序排列的数组）。脚本遍历所有正常窗口（`normalWindow && !minimized`），收集每个窗口的 `caption`、`resourceClass`、`frameGeometry`（x, y, width, height），然后通过 `callDBus()` 将 JSON 数据发送回应用。

4. **接收数据**：应用在 D-Bus 上注册了自己的服务（见 9.3），脚本调用该服务的 `receive_window_data` 方法将窗口列表传回。

5. **清理**：数据接收后，调用 `unloadScript(pluginName)` 卸载脚本，删除临时文件。

整个过程有 5 秒超时。超时后窗口吸附功能静默禁用。每次进入截图模式都会重新获取窗口列表。

### 9.3 应用自身的 D-Bus 服务

应用注册了自己的 D-Bus 服务，用于两个目的：

- **服务名**: `com.takeashot.service`
- **对象路径**: `/com/takeashot/Service`
- **接口**: `com.takeashot.Service`

暴露两个方法：

- `activate()` — 触发截图模式。用于单实例管理：第二个进程启动时发现服务名已被占用，就调用此方法激活已有实例，然后自行退出。
- `receive_window_data(json_str: string)` — 接收来自 KWin 脚本的窗口列表数据（见 9.2 步骤 4）。

单实例判断的依据是能否在 Session Bus 上成功注册 `com.takeashot.service` 这个服务名。注册失败说明已有实例在运行。

### 9.4 全局热键：evdev 原始输入

不使用桌面环境的快捷键机制，而是直接读取 Linux 内核的输入设备：

- 枚举 `/dev/input/` 下所有设备，筛选名称中包含 "keyboard" 的设备。
- 使用 `select()` 多路复用监听所有键盘设备的事件。
- 捕获 `KEY_PAUSE`（Pause 键）的按下事件（`value == 1`）。
- 设备断开时优雅处理（从监听列表中移除），不会崩溃。

选择 evdev 而非桌面快捷键 API 的原因是：在 Wayland 下，应用无法全局监听按键（这是 Wayland 安全模型的一部分）。evdev 绕过了这个限制，因为它直接读取内核级别的输入事件，但需要用户对 `/dev/input/` 设备有读取权限（通常通过 `input` 用户组实现）。

### 9.5 剪贴板

使用 Qt 的 `QGuiApplication.clipboard()` 将截图写入系统剪贴板。在 Wayland 下，Qt 内部通过 `wl_data_device` 协议与合成器通信，应用无需直接处理。

## 10. 关键设计决策

**仅支持 KDE Wayland**：不追求跨桌面兼容，专注于利用 KWin 的原生能力（截图接口、窗口信息、脚本 API），获得更好的集成体验。

**移动工具与标注编辑工具分离**：最初"选择/移动"是一个工具，既能操作选区也能操作标注，容易导致误操作。后来拆分为两个独立工具——移动选区工具不能触碰标注，标注编辑工具不能触碰选区，职责清晰。

**工具栏用 QML 实现**：最初用原生 Qt 控件搭建工具栏，后来改为 QML。QML 更适合声明式地描述 UI 布局和交互，也更容易调整视觉样式。

**窗口匹配用层叠顺序而非面积排序**：使用 KWin 的层叠顺序。层叠顺序更符合用户直觉——"我看到的最上面那个窗口"就是我想截的窗口。

**选区外部点击扩展而非清除**：在有选区状态下点击选区外部，不会清除当前选区重新开始，而是将选区扩展到包含点击位置。

**单实例**：同一时间只允许一个实例运行。第二次启动不会创建新进程，而是激活已有实例。避免多个截图遮罩叠加造成混乱。
